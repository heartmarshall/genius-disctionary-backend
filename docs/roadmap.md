## 1 - переход с RefCatalog на предзагруженные датасеты
1. Собираем датасет из поплуярных: книг, песен, сериалов и фильмов. Достаем из них слова, формируем список из 20-25 тысяч слов, которые встречюатся в них. 
2. Далее для каждого слова с помощью открытых датасетов подкачиваем: определение, транскрипцию, примеры, и другие поля, которые нужны для добавления слова в Ref словарь
3. Т.к качество примеров и определений из датасетов не самое лучшее, то далее будем реализовывать стратегию lazyLoad. Когда пользователи добавляют себе в словарь что-то из ref словаря, нужно делать пометку, что данное слово было использовано каким-то пользователем. Так оно попадает в очередь на обработку через LLM. Специальный скрипт должен раз в N часов или по требованию выкачивать все слова, которые попали в список подкачки lazyload и составлять batch запросы к LLM (промт + сам запрос), причем размер батча должен регулироваться. Далее эти запросы автоматически через пайплайн или в ручном режие администратором отправляются в LLM и реузльтаты выгружаются в виде JSON в дирректорию импорта. Должна быть возможность подтянуть эти резлуьтаты запросом или с помощью автообновления (скрипт проходится по дирректории и обновляет по json все сущности которые нашел)

Так мы сможем избежать необходимости сразу подготавливать датасет из 25 тысяч обработанных LLM слов.

Также нужно убедиться, что когда ref-сущность обновляется, то пользователи смогут увидеть все изменения у себя при открытии слов в своих словарях.

Видимо в ходе этого задания нужно будет реализовать админ-функционал. Специальная категория пользователей.

## 3 - новые режимы повторения
1. Стандартный режим - карточки, на которых написано слово, нужно открыть карточку и сказать правильно ли  ты угадал определение и перевод
    1. Next: добавить возможность добавлять подсказку к карточке. Чтобы пользователь мог добавлять подсказку, которая помогает ему вспомнить что это за слово.
    2. Давать возможность посмотреть определение слова. Но убедиться, что самого слова в определении нет.
    3. Перед открытием карточки давать возможность написать предложение с этим словом, чтобы оно запоминалось еще лучше. Вести историю таких предложений. Чтобы пользователь мог отслеживать как он развивался в этом.
2. Режим с определениями - показывается определение, нужно угадать что это за слово. 
3. Режим с "летящими словами". Для пк: справа экрана летят слова. пока они не достигли черты слева экрана, нужно написать их по английски. Если слово введено с ошибками, но близко к тому, как оно пишется реально, рядом со словом появляется то, которое нужно ввести, только в местах, где пользователь ошибся, показываются пропуски. Для смартфонов слова летят сверху вниз
