## 1 - переход с RefCatalog на предзагруженные датасеты
Сейчас RefCatalog Service работает по принципу **lazy loading** — данные попадают в каталог только когда пользователь запрашивает слово. В этот момент Composite Provider последовательно обращается к FreeDictionary API (определения, IPA, примеры) и Google Translate API (русские переводы), объединяет результаты и сохраняет в БД.

Этот подход имеет несколько проблем:

- **Зависимость от внешних API в runtime.** FreeDictionary — бесплатный сервис без SLA. Google Translate — платный. Любой сбой = пользователь не может добавить слово.
- **Холодный старт.** При первом запуске каталог пуст. Каждое слово требует двух HTTP-запросов.
- **Отсутствие метаданных.** Нет информации об уровне сложности (CEFR), частотности слова, семантических связях (синонимы, гиперонимы). Эти данные невозможно получить из FreeDictionary или Google Translate.
- **Ограниченное качество примеров.** FreeDictionary часто возвращает 0-1 пример на определение. Для обучения нужно больше.

**Идея:** предзагрузить каталог из свободных лингвистических датасетов. Это даёт ~20,000 готовых слов с определениями, произношением, переводами, примерами и семантическими связями — до того, как хоть один пользователь откроет приложение. Для слов за пределами seed-набора lazy loading через внешние API продолжит работать как раньше.
